# Тестовое задание на должность "Lead/Senior Backend Developer (Node.js / Go)" в GREEN-API

Учитывая что должность называется Backend Developer, при выполнении тестового задания, основные усилия были сосредоточены на Backend части. Кроме того, SDK предоставляемое GREEN-API, тоже не было использовано, чтобы показать максимальное владение используемыми инструментами.

## Стек технологий

Для реализации был выбран фреймворк NestJS. Для взаимодействия с GREEN-API был использован Axios, обернутый в модуль. Для логирования был использован winston. Для тестирования был использован jest. Для валидации и контрактов использован zod. Кроме того, все эндпоинты обернуты в swagger.

## Структура проекта

Проект состоит из следующих основных модулей:

- `message` - модуль для отправки сообщений и файлов
- `settings` - модуль для получения настроек и состояния инстанса
- `shared` - общие компоненты и утилиты

## Реализованные функции

1. **Модуль настроек (settings)**:
    - Получение текущих настроек аккаунта
    - Получение состояния аккаунта

2. **Модуль сообщений (message)**:
    - Отправка текстовых сообщений
    - Отправка файлов по URL

## Конфигурация

Настройки проекта вынесены в .env файл. Пример конфигурации можно найти в .env.sample.

## Тестирование

Для тестирования используется jest. Запуск тестов осуществляется командой `npm test`.

## Логирование

Для логирования используется winston, который позволяет вести логи в различных форматах и направлениях.

## Web-интерфейс

Для упрощения реализации web формы был использован JQuery и Tailwind.
